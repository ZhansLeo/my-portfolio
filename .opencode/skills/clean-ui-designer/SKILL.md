## 🌌 深色科技风格规范 (Dark Tech Style)

参考 hangkangfu.com 的视觉调性:深色沉浸感 + 渐变光晕 + 卡片化信息展示,但配色收窄为单一色系,避免撞色堆砌。

### 背景 (Background)
- 页面底色为深藏青紫色,如 `#0a0e27` 或 `#0f0a1e`,不用纯黑
- Hero 区域和主标题区块背后叠加柔和的渐变光晕(nebula 效果):用 `radial-gradient` 制造光斑,配合 `filter: blur(60px)` 让边缘模糊融入背景,光晕颜色控制在蓝紫色系(如 `rgba(99, 102, 241, 0.15)`)
- 光晕只用在 1-2 个关键区块(首屏、大标题处),其余区块保持纯色背景,避免每处都发光导致视觉疲劳
- 可选:背景空白处点缀稀疏的小圆点(装饰性粒子),用 `opacity` 控制透明度,营造星空感,数量要克制,不要铺满

### 导航栏 (Navigation)
- `position: sticky; top: 0`,半透明深色背景 + `backdrop-filter: blur(12px)`
- 布局:左侧 Logo(头像 + 品牌名) → 中间居中导航链接 → 右侧功能图标(如语言切换、主题切换)
- 导航栏本身不需要强调色,保持低调,让内容区域的渐变和光晕成为视觉焦点

### 标题文字 (Headings)
- 大字号标题(如 Hero 区的问候语)使用渐变文字效果:
```css
  background: linear-gradient(90deg, #6366f1, #a855f7);
  -webkit-background-clip: text;
  color: transparent;
```
- 渐变只用 2 个颜色过渡(蓝→紫),不要用三色以上的彩虹渐变
- 可以在标题前加一个 emoji 作为装饰符号(如 ✨🚩📷),但仅限大标题使用,不要每个小标题都加

### 卡片 (Cards)
- 卡片背景比页面底色稍浅一档(如底色 `#0a0e27` 对应卡片 `#12172e`),不用纯白或强对比
- 圆角适中,`border-radius: 12px-16px`
- 边框极淡或不用边框(`border: 1px solid rgba(255,255,255,0.06)` 或干脆不要边框)
- 卡片内部左侧或顶部搭配一个"图标色块"(icon chip):
  - 圆角方形容器(约 40-48px),内部放白色线性图标
  - **色块颜色统一使用同一色系的深浅变化**(例如都在蓝紫色系里取色:靛蓝 `#6366f1`、紫罗兰 `#8b5cf6`、深紫 `#7c3aed`、蓝灰 `#4f46e5`),不要用红、绿、橙、黄等完全不同色相堆在一起
  - 靠明度/饱和度差异区分不同卡片,而不是靠色相跳跃

### 按钮 (Buttons)
- 胶囊形状(`border-radius: 9999px`)
- 主按钮用渐变背景,统一同一种渐变方向和色系(如蓝→紫),不要一个按钮纯蓝、另一个纯紫两种不同渐变并存
- 次要按钮可以用同色系但更淡的版本,或者描边样式

### 引用/金句区块 (Quote Blocks)
- 左侧加一条细色条(2-4px 宽)做视觉锚点,颜色同样限定在蓝紫色系内取值,不要每个区块换一种完全不同的颜色
- 标题加粗,配一句斜体英文引言作为点缀,正文段落用稍浅的灰白色(如 `rgba(255,255,255,0.7)`)拉开与标题的层级

### 配色总原则(重点)
- 全站围绕**一个主色系(蓝紫色)**展开,允许在这个色系内有深浅、饱和度变化,但不引入红、橙、黄、绿等其他色相作为大面积色块
- 唯一允许"跳出主色系"的地方是最核心的品牌标题渐变(可以蓝紫之外再加一点点缀色,但整体仍要克制,不超过 2-3 色)
- 强调色(渐变、光晕、图标色块)全部统一色调后,整站会显得更统一、更有"设计感",而不是"每个模块换个颜色玩"的拼贴感

## ✨ "关于我" 板块:光感灵动风格 (Glowing & Alive)

不要用纯静态深色块 + 无装饰文字的极简排版,而是让整个区块"看起来在发光、有生命力"。

### 背景光晕(核心)
- 在整个"关于我"区块背后叠加 2-3 个大尺寸的柔和光斑(radial-gradient + blur 40-80px)
- 颜色统一在蓝紫色系内(如 靛蓝 #6366f1、紫罗兰 #8b5cf6、品红 #a855f7 三色轮换),不引入其他色相
- 光斑位置错落分布(左上、右下等),制造氛围感,不要居中对称摆放显得死板
- 可选:让光斑有极缓慢的位置/透明度漂移动画(CSS keyframes,周期 8-15s),制造"呼吸"感,幅度要小,不能让人分心

### 标题:发光渐变 + 装饰符号
- 主标题用蓝紫渐变文字(沿用之前的 background-clip: text 方案)
- 标题前加一个小型 emoji 或图标作为点缀(✨ 之类),不要省略这个细节——它是"活泼感"的重要来源
- 可选:给渐变文字加极轻微的 `text-shadow` 或用一层模糊的同色渐变文字叠在正文下方做发光晕染效果

### 人像/头图处理(如果有个人照片)
- 给照片加一层色彩叠加(用 CSS `mix-blend-mode` 或半透明渐变遮罩),让照片色调和背景光晕融为一体,而不是孤立的一张"证件照"感照片
- 照片容器圆角保持柔和(12-16px),可以让照片背后透出一点光晕溢出边缘的效果

### 按钮:实心渐变胶囊
- 恢复实心渐变按钮(蓝紫渐变),不要用纯描边或文字链接,要有"存在感"
- hover 时轻微发光增强(box-shadow 颜色加深、扩散)

### 正文文字
- 这一点可以保留上次的建议:段落间距舒展、关键短语加粗,但**不需要**极简等宽字体标签和纯手写倾斜标题这些"克制到冷淡"的元素
- 可以保留分节的编号标记(如 01 关于我),但让编号本身也带一点渐变色或发光效果,呼应整体氛围,而不是纯灰色的低调标签

### 与深色科技风格规范的关系
- 这份"光感灵动"规范优先级高于之前"极简编辑排版"那版,后者作废
- 配色依然遵守之前定的"蓝紫单一色系,不撞色"原则——灵动感来自"光的流动"和"渐变呼吸",不是靠堆砌多种颜色
